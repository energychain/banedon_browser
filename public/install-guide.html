Step 3: Open Chrome Extensions Page

Step 4: Enable Developer Mode

In the top right corner of the Extensions page, toggle on "Developer mode".

Note: Developer mode is required to install unpacked extensions.

Step 5: Load the Extension

Click the "Load unpacked" button that appears after enabling Developer mode.

Select the folder where you extracted the extension files (the folder containing manifest.json).

Step 6: Verify Installation

After loading, you should see the Browser Automation extension in your extensions list. The extension icon should appear in your browser toolbar.

Success! The extension is now installed and ready to use.

Step 7: Configure the Extension

Click on the extension icon in your toolbar to open the popup. You'll need to configure:

  • Server URL: http://localhost:3010
  • Session ID: Create a session first using the API

Creating a Session

Before connecting the extension, create a session using the REST API:

curl -X POST http://localhost:3010/api/sessions \ -H "Content-Type: application/json" \ -d '{"metadata":{"browser":"chrome","purpose":"testing"}}'

This will return a session ID that you can use in the extension.

Step 8: Test the Connection

After entering the server URL and session ID in the extension popup:

  1. Click "Connect" in the extension popup
  2. The status should change to "Connected"
  3. You can now send commands to control this browser tab

Troubleshooting

Extension won't load:

  • Make sure Developer mode is enabled
  • Check that you selected the correct folder (containing manifest.json)
  • Verify all files were extracted properly

Can't connect to server:

  • Verify the server URL is correct
  • Make sure the Browser Automation Service is running
  • Check that you have a valid session ID
  • Test the health endpoint: Health Check

Commands not working:

  • Ensure the extension is connected (green status)
  • Check the browser console for error messages
  • Verify the session ID is valid and active
